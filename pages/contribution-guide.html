Contributing Guide

Pixel Phantoms is a fully open-source, student-friendly project. Whether you're fixing a bug or designing a banner, your help makes our community stronger.

01

Getting Started

Follow these steps to set up the project locally on your machine.

1. Fork the Repository: Click the Fork button at the top-right of the GitHub repo.

2. Clone Your Fork

git clone https://github.com/sayeeg-11/Pixel_Phantoms.git
cd pixel-phantoms

3. Create a Feature Branch

git checkout -b feature-your-feature-name
02

Project Setup

Since this is a static website, no backend setup is required.

Requirements

Any code editor (VS Code recommended) and basic HTML/CSS/JS knowledge.

Run Locally

Open index.html directly or use the Live Server extension in VS Code.

03

What You Can Contribute

Code

  • Improve UI/UX & Animations
  • Fix responsiveness issues
  • Add new feature pages

Content

  • Update page text
  • Update club details
  • Improve documentation

Design

  • Add banners & logos
  • Create event posters
  • Update icons & images
04

Contribution Rules

Crucial: Do not commit node_modules or .vscode. Keep media files small.

Commit Message Guidelines

Add: New gallery section
Fix: Navbar alignment issue
Update: Committee details
Refactor: CSS for responsiveness
05

Submitting Your Work

git add .
git commit -m "Describe your change clearly"
git push origin feature-your-feature-name
Note: Always include before/after screenshots for any UI changes in your PR.
06

Code of Conduct

  • Be respectful and supportive
  • Help beginners
  • Keep discussions constructive
  • Follow open-source etiquette
  • Avoid spamming issues or PRs
